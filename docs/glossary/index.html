Glossary

artifact

When used as a UML keyword: A physical piece of information that is used or produced in software development, or by deployment and operation of a system. See [1], §19.5.1.

branch

In Git: A movable pointer to a commit. Can also be synonymous with codeline.

codeline

A sequence of versions of a codebase, often in the form of commits to a version control system. See [5].

composition root

A single, logical location in an application where modules are composed together. See [8], §4.1.

device

When used as a UML keyword: A node representing a physical computational resource on which artifacts may be deployed. See [1], §19.5.7.

execution environment

When used as a UML keyword: A node offering an execution environment for specific types of components deployed on it as executable artifacts. See [1], §19.5.8.

healthy

A codebase or branch is healthy if it builds successfully with few, if any, bugs. See [5].

keyword

In the context of UML: A textual annotation that serves to distinguish a particular UML concept from others sharing the same general graphical form. Keywords are always enclosed in guillemets (i.e., «keyword»). Not to be confused with UML stereotypes. See [1], annex C.

mainline

The codeline or branch that acts as the current state of the product. Usually called main or master. See [5] and [6].

node

In the context of UML: A computational resource upon which artifacts may be deployed. See [1], §19.4.

profile

In the context of UML: Extends UML with a collection of stereotypes. The standard profile specifies a set of predefined standard stereotypes. See [1], §12.3 for a description of profiles in general, and §22 for the standard profile.
